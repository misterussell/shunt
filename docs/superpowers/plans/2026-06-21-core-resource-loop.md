# Core Resource Loop Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a persisted Cred/Scrip/Heat resource model with one generation action ("Do a Job") and one spend action ("Lay Low"), displayed on a plain wireframe LiveView dashboard that replaces the generated Phoenix welcome page.

**Architecture:** A `Shunt.Players` context owns a singleton `Player` Ecto schema (one DB row, no auth). `ShuntWeb.DashboardLive` replaces the `/` route entirely, calling only context functions — never `Repo` or the schema directly. No background process, scheduler, or PubSub — every resource change is driven by a direct LiveView `handle_event` triggered by a button click.

**Tech Stack:** Elixir 1.19.5 / OTP 28, Phoenix 1.8.1, Phoenix LiveView 1.1, Ecto + Postgrex, Postgres 16 (via the project's existing `docker-compose.yml`).

## Global Constraints

- Per `docs/superpowers/specs/2026-06-21-core-resource-loop-design.md`: no auth, no theming, no passive/auto-tick generation, no Heat Events.
- Singleton player only — `Shunt.Players.get_or_create_player/0` is the only way to obtain a `%Player{}`.
- Balance constants (placeholder, not load-bearing): Do a Job = `+15 Scrip, +5 Cred, +10 Heat`. Lay Low = `-10 Cred, -20 Heat`, fails with `{:error, :insufficient_cred}` if `cred < 10`.
- Heat clamps to `[0, 100]`; Cred/Scrip floor at `0`.
- New UI code uses plain Tailwind utility classes only (no daisyUI `btn`/`card` classes) per `AGENTS.md` — the generated layout already uses daisyUI-style classes; don't touch that, just don't add more of it in new code.
- Postgres must already be running (`docker compose up -d` from the project root) before running `mix ecto.*` or `mix test`.

```elixir
  @job_scrip_gain 15
  @job_cred_gain 5
  @job_heat_gain 10

  def do_job(%Player{} = player) do
    player
    |> Ecto.Changeset.change(%{
      scrip: player.scrip + @job_scrip_gain,
      cred: player.cred + @job_cred_gain,
      heat: clamp_heat(player.heat + @job_heat_gain)
    })
    |> Repo.update()
  end

  defp clamp_heat(heat), do: heat |> max(0) |> min(100)
```

```elixir
  @lay_low_heat_reduction 20

  def lay_low(%Player{cred: cred}) when cred < @lay_low_cred_cost do
    {:error, :insufficient_cred}
  end

  def lay_low(%Player{} = player) do
    player
    |> Ecto.Changeset.change(%{
      cred: max(player.cred - @lay_low_cred_cost, 0),
      heat: clamp_heat(player.heat - @lay_low_heat_reduction)
    })
    |> Repo.update()
  end
```

```elixir
defmodule ShuntWeb.DashboardLive do
  use ShuntWeb, :live_view

  alias Shunt.Players

  def mount(_params, _session, socket) do
    {:ok, assign(socket, player: Players.get_or_create_player())}
  end

  def handle_event("do_job", _params, socket) do
    {:ok, player} = Players.do_job(socket.assigns.player)
    {:noreply, assign(socket, player: player)}
  end

  def handle_event("lay_low", _params, socket) do
    case Players.lay_low(socket.assigns.player) do
      {:ok, player} -> {:noreply, assign(socket, player: player)}
      {:error, :insufficient_cred} -> {:noreply, socket}
    end
  end

  def render(assigns) do
    ~H"""
    <Layouts.app flash={@flash}>
      <div class="space-y-6">
        <div class="border border-gray-300 rounded-lg p-4 space-y-2">
          <p id="resource-cred">Cred: {@player.cred}</p>
          <p id="resource-scrip">Scrip: {@player.scrip}</p>
          <div id="resource-heat">
            <p>Heat: {@player.heat}/100</p>
            <div class="w-full h-2 bg-gray-200 rounded">
              <div class="h-2 bg-red-500 rounded" style={"width: #{@player.heat}%"}></div>
            </div>
          </div>
        </div>

        <div class="flex gap-4">
          <button
            id="do-job-button"
            phx-click="do_job"
            class="px-4 py-2 rounded bg-blue-600 text-white hover:bg-blue-700"
          >
            Do a Job
          </button>
          <button
            id="lay-low-button"
            phx-click="lay_low"
            class="px-4 py-2 rounded bg-gray-600 text-white hover:bg-gray-700 disabled:opacity-50 disabled:cursor-not-allowed"
            disabled={@player.cred < 10}
          >
            Lay Low
          </button>
        </div>
      </div>
    </Layouts.app>
    """
  end
end
```
