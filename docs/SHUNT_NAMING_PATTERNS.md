# SHUNT NAMING PATTERNS

This document provides naming conventions for new content.

These are guidelines, not rules.

---

# Location Names

Locations should sound functional.

Formats:

* [Function] + [Location]
* [Infrastructure] + [Descriptor]
* [Physical Feature] + [Purpose]

Examples:

* Pipe Market
* Intake Row
* Relay Block
* Cooling Yard
* Service Spine
* Transfer Hall
* Maintenance Junction

Avoid:

* Neo Tokyo
* Cyber City
* Nexus Prime
* Alpha Sector

---

# District Names

Formats:

* [Infrastructure Name]
* [Worker Nickname]
* [Historical Usage]

Examples:

* Shunt 9
* Dryside
* The Intake
* Old Relay
* East Pipe

---

# Faction Names

Formats:

* [Occupation]
* [Tool]
* [Resource]
* [Place]

Examples:

* Wirekeepers
* Tunnel Union
* Scrap Court
* Relay Crew
* Pipehands
* Freight Syndicate

Avoid:

* Shadow Ravens
* Crimson Dragons
* Cyber Legion

---

# Technology Names

Technology names should be short.

Usually one word.

Examples:

* Ghostdeck
* Wirejack
* Signal Tap
* Relay Key
* Patchkit
* Splicer

Avoid:

* HyperNet X9000
* QuantumSync Ultra
* CyberLink Pro

---

# Event Names

Events should describe a situation.

Good:

* Broken Relay
* Missing Courier
* Tunnel Junkie
* Cold Signal
* Lost Shipment

Bad:

* Operation Nightfall
* Project Omega
* Digital Apocalypse

---

# NPC Names

NPC names should feel lived in.

Formats:

* First Name
* Nickname
* Occupational Nickname

Examples:

* Mara
* Vex
* Coil
* Switch
* Grease
* Docker
* Ash

Avoid excessive cyberpunk clichés.

---

# Skill Names

Skill names should be broad concepts.

Good:

* Ghostwork
* The Web

Avoid:

* Elite Hacker
* Cyber Mastery
* Shadow Protocol

---

# Naming Test

Before adding a name, ask:

1. Could a worker plausibly say this?
2. Does it sound useful rather than impressive?
3. Does it fit a salvaged future?
4. Would it still make sense twenty years later?

If the answer is yes to all four, the name is likely appropriate.
